**Obsidianのノートをフラッシュカードに。特別な構文は不要。個別のデッキを作成する必要もありません。**

ファイルに `#decks` のタグを付けるだけ。`##` の各見出しがカードの表面（Front）になり、その下のテキストが裏面（Back）になります。テーブル、画像の穴埋め（Image occlusion）、`==cloze==` のハイライトも同じように機能します。スケジュール管理は、最新の間隔反復アルゴリズムである「FSRS-6」によって行われます。

## Decks を使う理由

- **あなたのノートがすでにデッキです。** ファイルにタグを付ければ、選んだレベルの各見出しが表面になり、その下のテキストが裏面になります。Anki のように、ノートとカードを二重に作成する必要はありません。
- **4つのフォーマット、覚えるべき構文はなし。** 見出し、2列のテーブル、画像の穴埋め、そしてすでに使っているハイライトからの `==cloze==` （穴埋め）。
- **試験モード。** 選択式・入力式・穴埋め問題による採点付きの試験セッション。
- **FSRS ネイティブのスケジュール管理。** 3つのプロファイル（Standard / Intensive / Trained）、タグごとの記憶定着ターゲット、古い SM-2 のしがらみはゼロ。
- **アルゴリズムの微調整。** ワンクリックで、あなた自身の復習履歴に基づいて FSRS の重みをトレーニング。あなたの忘却曲線に合わせた最適なスケジュールを、すべてクライアント側（ローカル環境）で実現します。
- **真のマルチデバイス同期。** iCloud や Dropbox を通じてデータベースが自動的にマージ。スマホとPCの両方で復習でき、履歴が失われることはありません。
- **モバイル向け設計。** タッチ操作に最適化された復習UI、セーフエリア対応、毎日スマートフォンでテスト済み。

## 60秒クイックスタート

1. コミュニティプラグインから **Decks** をインストールし、有効化します。
2. 任意のノートを開きます。フロントマター、またはインラインタグとして `#decks` を追加します。
3. `##` の見出しを書き、その下に段落を書きます。カードの枚数分だけ繰り返します：

   ```markdown
   ---
   tags: [decks/english]
   ---

   ## 「Hola」とはどういう意味ですか？

   こんにちは（Hello）。

   ## 英語で「ありがとう」はどう言いますか？

   Thank you.
   ```

4. リボンにある **脳のアイコン** をクリックして Decks パネルを開きます。ファイルをクリックして、復習を開始します。

ファイル名がデッキ名になります。ノートを保存すると、カードは自動的に同期されます。

## カードのフォーマット

Decks は4つのカード作成方法をサポートしています。あなたのノートの書き方に合わせてお選びください。

<details>
<summary><b>見出し + 段落</b> — デフォルト形式。設定したレベル（デフォルトは H2）の各見出しが表面で、その下の本文が裏面になります。</summary>

```markdown
---
tags: [decks/english]
---

## 「Hola」とはどういう意味ですか？

こんにちは。

## 英語で「ありがとう」はどう言いますか？

Thank you.
```

ファイル名がデッキ名になります。見出しのレベルはプロファイルごとに設定可能です（デフォルトは H2）。設定したレベルより上の見出しはカードになりません。コンテキストとして各カードにパンくずリスト（例: `第1章 > セクション2`）として付加されます。

見出し+段落カードには、本文中の任意の場所に Obsidian コメント（`%%ヒント%%`）を書くか、末尾の `---` 区切りの後にテキストを置くことで、任意の**メモ**を追加できます。メモは復習中に必要に応じて（**N** キー）表示されます。

</details>

<details>
<summary><b>テーブル</b> — 2列の Markdown テーブル。オプションでノート列も追加できます。</summary>

```markdown
## 概念

| 質問                 | 答え                                         |
| -------------------- | -------------------------------------------- |
| 光合成とは何ですか？ | 植物が太陽光を変換するために使用するプロセス |
| 重力を定義しなさい   | 物体を互いに引き寄せる力                     |
```

- 1列目 = 表面、2列目 = 裏面。見出し行は無視されます。
- テーブルは必ず見出しの直下に置く必要があります（間に別の段落を含めないこと）。
- 3列目に「ノート」列を追加すると、復習中に（**N** キーを押して）ヒントや語呂合わせを表示できます。

</details>

<details>
<summary><b>穴埋め (Cloze deletions)</b> — テキストを <code>==テキスト==</code> でハイライトして空欄にします。</summary>

```markdown
## 太陽系

==太陽== は私たちの太陽系の中心にある恒星です。最も近い惑星は ==水星== で、最も大きな惑星は ==木星== です。
```

それぞれのハイライトが1枚のカードになります。復習中、アクティブな空欄は `[...]` と表示され、タップすると正解が表示されます。穴埋めはテーブルのセル内でも機能します。プロファイルごとに2つのコンテキストモード（他の穴埋めを隠す、または表示する）があります。デフォルトで有効です。

</details>

<details>
<summary><b>画像の穴埋め (Image occlusion)</b> — 画像の領域を隠し、その下にあるものを思い出します。2 つの方法：インタラクティブ（ボックスを描く）または番号付きリスト。</summary>

**インタラクティブ（推奨）。** コマンド **「カーソル位置に画像オクルージョンを作成」** を実行し、画像を選んで、エディターで画像上に直接ボックスを描きます。各ボックスに Markdown/LaTeX の答えを入力するか、画像に埋め込まれたラベルを隠すだけなら空欄のままにします。自己完結型の `decks-occlusion` コードブロックとして保存されます（座標はパーセントなので、どのデバイスでも拡大縮小します）：

````markdown
```decks-occlusion
image: "[[heart.png]]"
version: 2
masks:
  - id: m1
    x: 12.5
    y: 30
    w: 18
    h: 9.5
    answer: "左**心室**"
  - id: m2
    x: 55
    y: 22
    w: 14
    h: 8
    answer: ""
```
````

各ボックスが 1 枚のカードです。復習では、ボックスは表面で隠され、裏面で（答えとともに）表示されます。閲覧ビューでは、完全にラベル付けされた図が表示されます。ブロックの **編集** ボタンまたはフラッシュカードマネージャーからいつでも編集できます。

**番号付きリスト（シンプル）。** 埋め込み画像のあとに番号付きリストを置く方法も使えます：

```markdown
## 腕の骨

![[arm_bones.png]]

1. ==上腕骨==
2. ==橈骨 (とうこつ)==
3. ==尺骨 (しゃっこつ)==
```

リストの各項目が 1 枚のカードになります。画像が表面に表示され、一致する項目が裏面で空欄になります。

どちらも穴埋め (Cloze) 機能を利用しているため、プロファイルで Cloze を有効にし、ブロックは解析される見出しの下に置く必要があります。

</details>

<details>
<summary><b>その他：タイトル形式、リバースカード、カードごとのタグ</b></summary>

**タイトル形式** — ファイル名が表面に、ファイル全体が裏面になります。プロファイルで「Title（タイトル）」を見出しレベルとして設定してください。

**リバースカード（裏表反転）** — ファイルのフロントマターに `reverse: true` を追加すると、すべてのカードの裏表を反転させたコピーが自動生成されます。進捗はそれぞれの方向ごとに個別に追跡されます。

**カードごとのタグ** — 見出しに直接 `#タグ` を追加します（例：`## 光合成とは何ですか？ #植物 #高校`）。タグは表示される表面からは取り除かれ、復習中にチップとして表示され、テーブルの行やリバースカードにも引き継がれます。Vault 全体から特定のタグを持つカードを抽出する「フィルターデッキ」を作成できます。

</details>

### Canvas デッキ

Markdown ファイルの代わりに Obsidian の Canvas (`.canvas`) 上でカードを作成できます。設定したフォルダ内の各 canvas が 1 つのデッキになり、各テキストノードは上記と同じ 4 つのカード形式で解析されます。**設定 → Canvas デッキ** で設定します：フォルダとタグ（既定値 `#decks/canvas`）。復習中の「ソースに移動」は canvas を開き、該当のテキストノードにフォーカスします。初回インストール（またはアップグレード）時には、`Canvas decks/` フォルダ内に `Decks — Canvas はじめに.canvas` が自動的に作成されます。

**空間カード（Spatial cards）**：テキストノードをエッジで接続すると、各エッジが 1 枚のカードになります — 始点ノードが表面（質問）、終点ノードが裏面（答え）、エッジのラベルは任意のヒントです。チェーン（A → B → C）、1 対多、多対 1 のいずれも動作します。接続されていないノードは引き続き上記の 4 つの形式で解析されます。詳細は **[docs/CANVAS_DECKS.md](docs/CANVAS_DECKS.md)** を参照。

![Canvas Spatial Cards Demo](./canvas_spatial_cards_demo.gif)

## テンプレート

一度だけ作るカードデザインで、表の各行をレンダリングします。HTML/CSS または Markdown で記述し、`{{Column}}`
プレースホルダーを置き、タグで表に紐付けます — 1 つのテンプレートが、一致するすべての行にスタイルを適用します。

```decks-html-front
<ruby>{{Word}}<rt>{{Reading}}</rt></ruby>
```

**設定 → テンプレート** でフォルダを指定し、テンプレートファイルと表の見出しに同じタグを付けるだけです。テンプレートは
HTML または Markdown で表面／裏面／ノートの各面に対応し、サニタイズされたテーマ対応のサンドボックス内でレンダリングされ、
レイアウトを完全に制御できる CSS 変数（`--padding`、`--align`、`--bg` など）を提供します — 読みやすいカードから、端まで
広がるカスタムデザインまで。一致するテンプレートがない表は、これまでどおり通常の列を使います。

詳しいガイドと例は **[docs/TEMPLATES.md](docs/TEMPLATES.md)** を参照してください。

## 試験デッキ

デッキを採点付きの試験として実施できます：抽出された一組の問題に 1 回のセッションで好きな順序で解答し、最後に結果の内訳 — さらに任意で制限時間と合格点 — が表示されます。試験はプロファイルごとに有効化され、カードの記述フォーマットが 1 つ追加されます：見出しの下にタスクリストを書くと選択式カードになります。

```markdown
## 希ガスである元素はどれですか？

- [ ] 酸素
- [x] アルゴン
- [ ] 窒素
```

`- [x]` が正解の選択肢を示します。複数チェックすると複数選択問題になります。

- デッキタグの `exams` サブタグ（デフォルトは `#decks/exams`）をノートに付けると、プリインストールされた **Exams** プロファイルが使われます。あるいは、任意のプロファイルで **Exam questions** をオンにします。
- デッキのメニュー（**⋮ → Start exam**）から、または試験デッキをクリックして開始します。セットアップダイアログに問題数が表示され、試験の設定を調整できます。
- 選択式のほか、見出し+答えのカードと表の行は入力式の問題として出題され、穴埋めカードはハイライト部分を入力欄にした文が表示されます。
- 入力した答えの採点は、完全一致、小さなタイプミスの許容、または自己採点から選べます。試験のデフォルト（問題数、制限時間、合格点、シャッフル、フィードバックのタイミング、選択肢ラベル）はプロファイルに保存されます。
- 完了した試験はプラグインのデータベースに保存され、デバイス間でマージされ、バックアップにも含まれます。

初回インストール時に、すべての問題形式を示す「Demo exam」デッキが作成されます（**「Create demo exam deck」** コマンドでも作成できます）。

記述ルールの詳細は **[docs/EXAM_DECKS.md](docs/EXAM_DECKS.md)** を参照してください。

## パーソナライズされたスケジュール管理

FSRS は初期状態でも十分に機能するデフォルト設定が備わっています。約100回の復習が蓄積されると、あなた自身の復習履歴に合わせてアルゴリズムの 21 個の重みをトレーニングし、特定の忘却曲線に合わせたカードのスケジュールを得ることができます。デスクトップ版 Anki が行っていることと同じですが、サーバーやデータ送信を介さず、すべてクライアント側で実行されます。

## Spaced Repetition から移行する

すでに **Spaced Repetition** プラグインを使っていますか？ **カードもレビュー履歴も失うことなく** Decks に切り替えられます。レビューを中断したところからそのまま続けられます。

デッキパネルのツールバー（キューブのアイコン）から移行ツールを開くか、**「Migrate from Spaced Repetition plugin」** コマンドを実行してください。

**元のノートには一切手を加えません。** 移行は追加型です。選択した出力先フォルダに（構造をミラーしながら）新しいファイルを書き込み、ソースノートはそのままにします。移行ツールを再実行すると、生成したファイルが上書きされるだけです。

**動作の仕組み**

1. **スキャンするソースフォルダを選び**（空のままにするとボールト全体をスキャンします）、出力先フォルダを指定します。Decks は従来のカードを含むすべてのノートを見つけます。単一行（`Front :: Back`）、反転（`Front ::: Back`）、複数行（`?` / `??`）、穴埋め（`==…==` / `{{…}}`）、そしてノート全体のレビュー（`#review` タグ）です。
2. **各ノートは 2 つのきれいなファイルに分割されます。** **フラッシュカードデッキ**（`<ノート> (フラッシュカード)`）が抽出されたカードを標準の Decks 形式で保持し、**読みやすいノート**が本文を保持します。カード構文は通常のテキストへと*脱糖*されます（`::` / `:::` は " — " になり、`?` / `??` は問いと答えを結合し、`==…==` / `{{…}}` の穴埋めはその答えに復元されます）。設定したセパレーターが尊重されるため、カスタマイズしていても機能します。何も取り除かれず、読み用ノートは完全なまま保たれます。
3. **ファイルはフロントマターで相互リンクされます。** 読みやすいノートにはデッキを指す `フラッシュカード` プロパティと、元のノートを指す `元のノート` プロパティが付きます。デッキにも `元のノート` プロパティが付きます。ノートがすでにそれらのプロパティ名のいずれかを使っている場合、移行ツールは重複を作らず上書きします。タグは保持され、従来のベースタグ（例: `#flashcards`）は設定した Decks のタグに変換されます。
4. **反転カードは 2 枚のカードになります。** `Front ::: Back` のカードは、**同じ**デッキファイル内で順方向のカードと入れ替えたカードに展開されるため、各方向が独立してスケジュールされます。
5. **入れ子構造はコンテキストに平坦化されます。** SR は祖先の見出しや入れ子のリスト箇条書きをカードのコンテキストとして扱います。Decks はそのパス全体をカードの表面に取り込みます。例えば深く入れ子になった `Function :: Powerhouse` は `Cell Anatomy > Mitochondria > … > Function` になり、プロファイルの単一の見出しレベルでレンダリングされます（単独のノートタイトル H1 は省略されます）。プリインストールされた **Heading 1–6** プロファイルで任意のレベルを選べます。
6. **スマートな自動ルーティングが最適なレイアウトを選びます。** 短い単一行のカードはコンパクトな**テーブル**の行になります（語彙で延々とスクロールせずに済みます）。コードブロック、リスト、数式を含む複数行のカードは、書式が保たれるように**見出し**になります。ダイアログで*すべて見出し*または*すべてテーブル*に上書きできます。
7. **ノート全体のレビューも移行されます。** ノート全体としてレビューしていたもの（`#review` タグ）は、専用の `…/review` プロファイルの下で Decks の**タイトルモード**カード（ファイル名 = 表面、ノート全体 = 裏面）になります。そのスケジュールはノートの `sr-*` フロントマターまたはファイル末尾のマーカーから読み取られます。
8. **スケジュール状態は FSRS-6 に変換されます。** Decks は従来の `<!--SR:-->` メタデータ（SM-2 の `due, interval, ease`、またはすでに FSRS のもの）を読み取り、stability / difficulty / due の状態にマッピングします。反転カードは、元のプラグインが保存していたとおり、**2 つの別個の**履歴（読み取り用と想起用）を保持します。
9. **移行されたすべてのカードにレビューログが書き込まれます。** そのため、カードが Decks に現れた瞬間から、正しい日付に正しい間隔ですでに期限が来ています。やり直すのではなく、再開できます。

ダイアログでプロファイルを選んでください（またはデフォルトを使用）。その見出しレベルとスケジュール設定が移行されたデッキに適用されます。

## Anki から移行する

Anki から乗り換えますか？コレクション全体を Decks に取り込めます——**カード・メディア・学習履歴を失うことなく**——そして FSRS-6 で学習を続けられます。

Anki で、デッキ（またはコレクション全体）を **`.apkg`** としてエクスポートします（**ファイル → エクスポート**、形式は *Anki デッキパッケージ*、**メディアを含める**と**スケジュール情報を含める**にチェック）。次にデッキパネルのツールバーからインポーターを開くか、**「Import from Anki」** コマンドを実行し、ファイルと保存先フォルダーを選んでインポートします。旧形式・新形式（圧縮）どちらの `.apkg` エクスポートも利用できます。

**Anki のコレクションには一切手を加えません。** インポートは追加方式です。選んだ保存先フォルダー（`#decks/anki` タグの下に入れ子）に新しいファイルを書き込み、元の `.apkg` はそのまま残します。同じファイルを再インポートすると、生成済みのファイルが上書きされメディアが更新されます——いつでも再実行できます。

**仕組み**

1. **`.apkg` と保存先フォルダーを選びます。** Decks はそれをメモリ内で展開し、埋め込まれた Anki コレクション（旧形式または新しい圧縮形式）を読み取り、参照されるすべてのメディアファイルを保管庫の `media/` フォルダーへコピーします。元の Anki デッキ階層（`親::子`）はフォルダーとして保持されます。
2. **どのノートタイプも見やすい Decks カードになります。** 基本ノートはコンパクトな**テーブル**と**見出し**を自動で切り替えます。**cloze** の空所は `==…==` ハイライトになります——`$…$` MathJax 内の cloze も含みます。**複数フィールド／テンプレート**ノートには自動生成テンプレートが付き、Anki の**画像隠し**カードは Decks ネイティブの隠しとして取り込まれます。
3. **メディア・数式・タグも引き継がれます。** 音声と画像は埋め込まれ、レビュー時に再生／表示されます。画像は元のサイズを保ち、LaTeX/MathJax は保持され、Anki のタグはグループ化され読みやすいセクションに並べ替えられます。
4. **スケジュール状態は FSRS-6 に変換されます。** 各カードの期限日・間隔・易しさ——さらに Anki の全レビュー履歴——が安定度／難易度／期限の状態にマッピングされ、レビューログとして書き込まれます。そのためカードは**正しい日付に正しい間隔ですでに期限を迎えた**状態で現れます。やり直しではなく、続きから再開できます。
5. **大きくメディアの多いデッキも快適です。** 大きなデッキは、カード枚数とメディア埋め込み数の両方に基づいて、上限付きのサブフォルダー分割ファイルへ自動的に分割されます。そのため数千の音声クリップを含むデッキでも Obsidian で素早く開けます。小さなデッキは単一ファイルのままです。
6. **進行状況を確認できます。** 進捗バーが各フェーズ——コレクションの読み込み、デッキの書き込み、メディアのコピー、同期、レビュー履歴のインポート——を追跡するので、大きなインポートでも固まって見えることはありません。

ダイアログでプロファイルを選びます（または既定のまま）——その見出しレベルとスケジュール設定が、`#decks/anki` タグの下に入れ子になるインポート済みデッキに適用されます。

## AI アシスト（任意）

任意の AI 機能で、**設定 → AI でプロバイダーのキーを追加するまでは無効**です:

- **生成** — プロンプト（および任意のメモ/画像）からカードを作成し、新規ファイルまたは既存のデッキに、見出し+段落・テーブル・キャンバスのいずれかとして保存します。
- カード単体や選択全体を**リファクタリング**、または 1 枚のカードを複数に**分割**します。各変更は適用前に確認できます。

OpenAI、Anthropic（Claude）、Google（Gemini）、または OpenAI 互換/ローカルのエンドポイントのキーをご自身で用意してください。キーはローカルの `ai-keys.json` に保存され、`data.json` には決して書き込まれないため、同期で端末外に出ることはありません。操作を実行しない限りプロバイダーには何も送信されません。各リクエストには、Decks の仕組みに関する組み込みプロンプト、あなたの指示、その操作の内容のみが含まれます。
