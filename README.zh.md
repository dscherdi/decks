**将您的 Obsidian 笔记变成抽认卡（Flashcards）。无需特殊语法，无需建立单独的牌组。**

只需用 `#decks` 标记一个文件。每个 `##` 标题都会变成卡片的正面；下方的文字会变成背面。表格、图片遮挡和 `==填空==` 高亮显示也能以同样的方式工作。卡片复习调度由 FSRS-6 —— 现代间隔重复算法来处理。

## 为什么选择 Decks

- **您的笔记就是牌组。** 标记一个文件，所选层级的每个标题都会变成正面，下方的文字变成背面。如果您习惯使用 Anki，那么在这里您不需要重复编写两次。
- **四种格式，无需学习语法。** 标题、双列表格、图片遮挡以及您已经在使用的 `==填空==` 高亮。
- **FSRS 原生调度。** 三种配置文件（标准 / 强化 / 已训练），可按标签设定记忆保留目标，彻底告别旧版 SM-2 算法。
- **算法微调。** 一键优化器可根据您自己的复习历史训练 FSRS 权重 —— 为您的遗忘曲线提供更好的复习调度，全部在本地客户端完成。
- **真正的多设备同步。** 数据库通过 iCloud/Dropbox 自动合并 —— 在手机和桌面上复习，绝不丢失历史记录。
- **专为移动端打造。** 触控优化的复习界面，适配安全区域 (safe-area)，每天都在手机上进行测试。

## 60秒快速入门

1. 从社区插件库安装 **Decks** 并启用。
2. 打开任意笔记。在其 frontmatter 或正文内联标签中添加 `#decks`。
3. 写一个 `##` 标题，然后在其下方写一段正文。需要多少张卡片就重复多少次：

   ```markdown
   ---
   tags: [decks/english]
   ---

   ## "Hola" 是什么意思？

   你好 (Hello)。

   ## 怎么用英语说 "谢谢"？

   Thank you.
   ```

4. 点击侧边栏中的 **大脑图标** 打开 Decks 面板。点击您的文件。开始复习。

文件名将成为牌组名称。卡片会在您保存笔记时自动同步。

## 卡片格式

Decks 支持四种编写卡片的方式。选择最适合您记录笔记习惯的一种。

<details>
<summary><b>标题 + 段落</b> — 默认格式。配置层级（默认 H2）的每个标题都是正面，正文下方是背面。</summary>

```markdown
---
tags: [decks/english]
---

## "Hola" 是什么意思？

你好。

## 怎么用英语说 "谢谢"？

Thank you.
```

文件名即为牌组名称。标题级别可以按配置文件进行更改（默认 H2）。高于所配置级别的标题不会变成卡片 —— 它们会作为面包屑路径（例如 `第 1 章 > 第 2 节`）附加到每张卡片上以提供上下文。

可为「标题+段落」卡片添加可选**备注**：在正文任意位置写一条 Obsidian 注释（`%%提示%%`），或在结尾的 `---` 分隔线之后书写。复习时按需（**N** 键）显示备注。

</details>

<details>
<summary><b>表格</b> — 两列 Markdown 表格，带有一个可选的笔记列。</summary>

```markdown
## 核心概念

| 问题             | 答案                   |
| ---------------- | ---------------------- |
| 什么是光合作用？ | 植物用来转化阳光的过程 |
| 解释一下重力     | 吸引物体相互靠近的力   |
```

- 第一列 = 正面，第二列 = 背面。表头行将被忽略。
- 表格必须直接位于标题下方（内部不能有其他段落）。
- 添加第三个“笔记”列，用于在复习期间通过按 **N** 键切换显示的提示或记忆法。

</details>

<details>
<summary><b>填空 (Cloze)</b> — 使用 <code>==文本==</code> 高亮语法来挖空。</summary>

```markdown
## 太阳系

==太阳==是我们太阳系中心的恒星。距离最近的行星是==水星==，最大的行星是==木星==。
```

每个高亮都会成为一张独立的卡片。在复习时，活动的填空将显示为 `[...]`；点击即可显示。填空也可以在表格单元格内工作。每个配置文件支持两种上下文模式（隐藏或显示其他填空）。默认启用。

</details>

<details>
<summary><b>图片遮挡</b> — 遮挡图片的区域并回忆其下的内容。两种方式：交互式（绘制方框）或编号列表。</summary>

**交互式（推荐）。** 运行命令 **“在光标处创建图像遮挡”**，选择一张图片，然后直接在编辑器中于图片上绘制方框。为每个方框输入 Markdown/LaTeX 答案，或留空以仅遮挡图片中已有的标签。它会保存为一个自包含的 `decks-occlusion` 代码块（坐标为百分比，因此可在任何设备上自适应）：

````markdown
```decks-occlusion
image: "[[heart.png]]"
version: 2
masks:
  - id: m1
    x: 12.5
    y: 30
    w: 18
    h: 9.5
    answer: "左**心室**"
  - id: m2
    x: 55
    y: 22
    w: 14
    h: 8
    answer: ""
```
````

每个方框都是一张卡片。复习时，方框在正面被遮挡，在背面（连同答案）被揭示；在阅读视图中你会看到完整标注的图示。可随时通过代码块的**编辑**按钮或卡片管理器进行编辑。

**编号列表（简单）。** 嵌入图片后跟一个编号列表同样有效：

```markdown
## 手臂骨骼

![[arm_bones.png]]

1. ==肱骨==
2. ==桡骨==
3. ==尺骨==
```

列表中的每个项目都是一张卡片；图片显示在正面，匹配的项目在背面被挖空。

两者都建立在填空功能之上，因此配置文件中必须启用填空，且代码块必须位于被解析的标题之下。

</details>

<details>
<summary><b>更多：标题格式，翻转卡片，单卡标签</b></summary>

**标题格式** — 文件名成为正面，整个文件成为背面。在您的配置文件中将“标题”设置为标题级别。

**翻转卡片** — 将 `reverse: true` 添加到文件的 frontmatter 中，以自动生成每张卡片的翻转副本。复习进度按方向分别跟踪。

**单卡标签** — 直接在标题中添加 `#标签`（例如，`## 什么是光合作用？ #植物 #高中`）。标签会从显示的正面剥离，在复习期间作为标签片显示，并可由表格行和翻转卡片继承。利用它可以建立“过滤牌组”，将整个库中包含特定标签的所有卡片提取出来。

</details>

### Canvas 牌组

在 Obsidian Canvas (`.canvas`) 文件中编写卡片，而不是 Markdown 文件。已配置文件夹中的每个 canvas 都会成为一个牌组；每个文本节点都会按照上述相同的四种卡片格式进行解析。在 **设置 → Canvas 牌组** 中配置：文件夹和标签（默认 `#decks/canvas`）。复习中点击「跳转到源」会打开 canvas 并聚焦到源文本节点。首次安装（或升级）时会在 `Canvas decks/` 文件夹中自动创建一份 `Decks — Canvas 入门.canvas`。

**空间卡片（Spatial cards）**：用连线连接文本节点，每条连线都会变成一张卡片——起始节点是正面（问题），终点节点是反面（答案），连线上的标签作为提示。支持链式连接（A → B → C）、一对多、多对一；没有连线的节点仍按四种格式解析。详情见 **[docs/CANVAS_DECKS.md](docs/CANVAS_DECKS.md)**。

![Canvas Spatial Cards Demo](./canvas_spatial_cards_demo.gif)

## 模板

用你只需设计一次的卡片样式来渲染表格的每一行。用 HTML/CSS 或 Markdown 编写，放入 `{{Column}}` 占位符，再用标签把它
绑定到表格——一个模板即可为每一个匹配的行设置样式。

```decks-html-front
<ruby>{{Word}}<rt>{{Reading}}</rt></ruby>
```

在 **设置 → 模板** 中指定一个文件夹，给模板文件和表格的标题打上相同的标签即可。模板支持 HTML 或 Markdown 的正面/背面/笔记
三个面，在经过净化、感知主题的沙箱中渲染，并提供 CSS 变量（`--padding`、`--align`、`--bg` 等）以完全掌控布局——从舒适的
阅读卡片到铺满整张卡片的自定义设计。没有匹配模板的表格仍使用普通的列。

完整指南和示例见 **[docs/TEMPLATES.md](docs/TEMPLATES.md)**。

## 个性化调度

FSRS 附带了开箱即用的合理默认设置。一旦您积累了约 100 次复习记录，就可以根据您自己的复习历史来训练算法的 21 个权重，并获得为您特定遗忘曲线量身定制的卡片调度安排 —— 这与 Anki 桌面版的作用一样，但完全在本地客户端运行，没有服务器，也没有数据遥测。

## 从 Spaced Repetition 迁移

已经在使用 **Spaced Repetition** 插件了吗？你可以切换到 Decks，**不会丢失你的卡片或复习记录**——并且可以从上次中断的地方继续复习。

从牌组面板工具栏（立方体图标）打开迁移工具，或运行 **“从 Spaced Repetition 插件迁移”** 命令。

**你的原始笔记永远不会被改动。** 迁移是增量式的：它会把新文件写入你选择的目标文件夹（镜像你的结构），并让你的源笔记保持原样。重新运行迁移工具只会覆盖它生成的文件。

**工作原理**

1. **选择一个要扫描的源文件夹**（或留空以扫描整个仓库），以及一个用于输出的目标文件夹。Decks 会找到每一篇带有旧版卡片的笔记——单行（`Front :: Back`）、反向（`Front ::: Back`）、多行（`?` / `??`）、填空（`==…==` / `{{…}}`）——以及整篇笔记复习（`#review` 标签）。
2. **每篇笔记会被拆分成两个干净的文件。** 一个**闪卡牌组**（`<笔记> (闪卡)`）以标准 Decks 格式保存提取出的卡片，一篇**可读笔记**保留正文——卡片语法会被*去糖化*为普通文本（`::` / `:::` 变成 “ — ”，`?` / `??` 把问题和答案连接起来，`==…==` / `{{…}}` 填空会还原为它们的答案）。你配置的分隔符会被遵循，因此即使你自定义过它们也能正常工作。不会删除任何内容——你的阅读笔记保持完整。
3. **这些文件会在 frontmatter 中相互链接。** 可读笔记会获得一个指向其牌组的 `闪卡` 属性，以及一个指回原文的 `来源笔记` 属性；牌组也会获得一个 `来源笔记` 属性。如果某篇笔记已经使用了其中一个属性名，迁移工具会覆盖它而不是创建重复项。你的标签会被保留——旧版基础标签（例如 `#flashcards`）会被转换为你配置的 Decks 标签。
4. **反向卡片会变成两张卡片。** 一张 `Front ::: Back` 卡片会在**同一个**牌组文件中展开为一张正向卡片和一张交换卡片，因此每个方向都会被独立排程。
5. **嵌套结构会被展平为上下文。** SR 把祖先标题和嵌套列表项视为卡片的上下文。Decks 会把整条路径捕获到卡片的正面——例如一个深度嵌套的 `Function :: Powerhouse` 会变成 `Cell Anatomy > Mitochondria > … > Function`——并在你配置档的单一标题级别下渲染（单独的笔记标题 H1 会被省略）。可通过预装的 **Heading 1–6** 配置档选择任意级别。
6. **智能自动路由会选择最佳布局。** 较短的单行卡片会变成紧凑**表格**中的行（词汇表无需无尽滚动）；多行卡片——含有代码块、列表或公式——会变成**标题**，以便保留其格式。你可以在对话框中将其覆盖为*全部标题*或*全部表格*。
7. **整篇笔记复习也会被迁移。** 你作为整篇复习的笔记（`#review` 标签）会在一个专用的 `…/review` 配置档下变成 Decks **标题模式**卡片（文件名 = 正面，整篇笔记 = 背面）。它们的排程从笔记的 `sr-*` frontmatter 或文件末尾标记中读取。
8. **你的排程状态会被转换为 FSRS-6。** Decks 会读取旧版 `<!--SR:-->` 元数据——SM-2（`due, interval, ease`）或已是 FSRS——并将其映射为稳定度/难度/到期状态。反向卡片会保留**两段独立的**记录（阅读与回忆），与原插件存储的方式完全一致。
9. **每张迁移的卡片都会写入一条复习记录**，因此卡片一出现在 Decks 中，它们就已经在正确的日期以正确的间隔到期了——你是继续，而不是重新开始。

在对话框中选择一个配置档（或使用默认值）——它的标题级别和排程设置会应用到迁移后的牌组。

## 从 Anki 迁移

正从 Anki 切换过来？你可以把整个收藏导入 Decks——**不丢失卡片、媒体或复习记录**——并继续使用 FSRS-6 学习。

在 Anki 中，将你的牌组（或整个收藏）导出为 **`.apkg`**（**文件 → 导出**，格式为 *Anki 牌组包*，勾选**包含媒体**和**包含调度信息**）。然后从牌组面板工具栏打开导入器，或运行 **“Import from Anki”** 命令，选择文件和目标文件夹并导入。新旧（压缩）`.apkg` 导出均可使用。

**你的 Anki 收藏永远不会被改动。** 导入是增量式的：它会把新文件写入你选择的目标文件夹（嵌套在 `#decks/anki` 标签下），并保持源 `.apkg` 原样。重新导入同一文件会覆盖它生成的文件并刷新其媒体——因此你随时可以重新运行。

**工作原理**

1. **选择 `.apkg` 和目标文件夹。** Decks 在内存中解压它，读取内嵌的 Anki 收藏（旧格式或较新的压缩格式），并把每个引用的媒体文件复制到你库中的 `media/` 文件夹。原始 Anki 牌组层级（`父::子`）以文件夹形式保留。
2. **每种笔记类型都变成干净的 Decks 卡片。** 基础笔记会在紧凑的**表格**与**标题**之间自动切换；**cloze** 挖空变为 `==…==` 高亮——包括 `$…$` MathJax 内的 cloze；**多字段／模板**笔记会获得自动生成的模板；Anki 的**图像遮挡**卡片会转换为 Decks 原生遮挡。
3. **媒体、公式和标签一并迁移。** 音频和图像会被嵌入，并在复习时播放／渲染；图像保持原始大小；LaTeX/MathJax 得到保留；你的 Anki 标签会被分组并排序为易读的小节。
4. **你的调度状态会被转换为 FSRS-6。** 每张卡片的到期日、间隔和易度——以及它完整的 Anki 复习记录——都会映射为稳定度／难度／到期状态并写为复习日志，因此卡片会**在正确的日期以正确的间隔已然到期**。你是继续，而不是重新开始。
5. **大型、媒体密集的牌组依然流畅。** 大牌组会自动拆分为受限的、按子文件夹归类的文件——同时按卡片数量和媒体嵌入数量——这样即使有数千段音频的牌组，在 Obsidian 中也能快速打开。较小的牌组保持为单个文件。
6. **你能看到进度。** 进度条会跟踪每个阶段——读取收藏、写入牌组、复制媒体、同步以及导入复习记录——因此即便是大型导入也绝不会看起来卡住。

在对话框中选择一个配置文件（或使用默认）——其标题级别和调度设置会应用到导入的牌组，这些牌组嵌套在 `#decks/anki` 标签下。

## AI 辅助（可选）

可选的 AI 功能，**在你于「设置 → AI」中添加提供商密钥之前处于关闭状态**：

- **生成** —— 根据提示（以及可选的笔记/图片）生成卡片，并以「标题+段落」、表格或白板的形式保存到新文件或已有卡组。

自带 OpenAI、Anthropic（Claude）、Google（Gemini）或兼容 OpenAI/本地端点的密钥。密钥保存在本地的 `ai-keys.json` 中，绝不写入 `data.json`，因此不会通过同步离开你的设备。除非你触发某个操作，否则不会向提供商发送任何内容；每个请求只包含关于 Decks 工作方式的内置提示、你的指令以及该操作的内容。
