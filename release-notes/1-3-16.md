### Changes

**Database migration**

- Fixed migration breaking databases when upgrading from older schema versions
- Column renames handled gracefully (e.g., `old_due_date` to `old_due_at`)
- Migration failure now rolls back and creates a fresh database with a notice to restore from backup
- Only review data is migrated; decks and flashcards are recreated from vault files on sync

**Backup restore**

- Fixed "table review_logs has no column named new_due_date" error when restoring old backups
- Backup restore now dynamically reads schemas and maps old column names to current ones

**Multi-device sync**

- Profiles and tag mappings are now merged during disk sync across devices

**Deck config modal**

- Profile dropdown now updates when switching between tags to show the currently mapped profile
- Tag dropdown shows full hierarchy (e.g., `#flashcards`, `#flashcards/math`, `#flashcards/math/calculus`) so profiles can be applied at any level
- Saving config changes triggers a full sync to re-parse flashcards with updated profile settings

**Profiles**

- Changing profile settings now triggers a full sync so flashcards are re-parsed with updated header level and other profile options
