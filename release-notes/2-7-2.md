### Decks 2.7.2 — exams open everywhere, and read better

This release makes exam decks start from every deck list, improves how multiple-choice cards look in review, fits the exam screens to mobile, and fixes a profile toggle that could affect the wrong profile.

#### **Exam decks now start from the deck list popup**

Opening the deck list from the ribbon icon or the command showed exam decks, but clicking one started a normal review instead of an exam, and the "Start exam" menu entry was missing there. Exam decks now start an exam from that list too — by clicking the deck or using its ⋮ menu — matching the sidebar.

#### **Multiple-choice cards are easier to read in review**

When a multiple-choice card comes up in a normal review session, the options now stack beneath the question instead of beside it. Showing the answer highlights the correct option in green, including the correct answer you didn't pick, so it's clear what the right choice was.

#### **Exam screens fit mobile**

The exam screen and its setup dialog now go full-screen on phones, the same as the review screen.

#### **Fixed: turning on exam questions could affect the wrong profile**

Enabling "Exam questions" runs a quick check for already-reviewed cards that would change type before applying the setting. If you switched to a different profile while that check was still running, the setting could end up applied to whichever profile you'd switched to instead of the one you toggled it on — enabling exam parsing on a profile you never touched. The toggle now only ever applies to the profile it was changed on. If a profile you didn't intend to use for exams has "Exam questions" turned on, turn it back off in the profiles manager.
