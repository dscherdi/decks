### Decks 2.8.3 — deck tree fixes and polish

This release fixes and refines the unified deck tree introduced in 2.8.2.

#### **Fixed: rows now nest correctly**

Files and folders at the same level line up, and a folder's children are indented one step further in. Previously rows without a chevron (files and leaf tags) sat a notch to the left of their folder siblings, so the nesting was hard to read. Every row also shows an icon now — a document icon for file decks, a distinct icon for canvas decks.

#### **Collapse branches while searching**

Typing in the filter still expands matches so you can see them, but you can now fold any branch back up during a search without it snapping open again. Clearing the search restores your normal expanded/collapsed state.

#### **Fixed: folders no longer show a count with nothing inside**

A folder whose only deck was pinned kept a New/Due count but expanded to nothing, because the deck had floated up to the Pinned block. Such empty folders are now hidden; the deck stays in the Pinned block.

#### **Flatter, more compact list**

Rows, the header, and the filter bar use tighter, even spacing, and clicking a deck, folder, section, or the review-activity toggle no longer shows a focus outline or a press bounce.
