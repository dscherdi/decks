### Decks 2.7.1 — the Exams profile can be saved again

This release fixes three issues in the exam decks feature introduced in 2.7.0.

#### **Fixed: saving the Exams profile rejected its own defaults**

The preinstalled Exams profile sets "New cards per day" to 0 so exam questions never enter your normal reviews as new cards. The profile editor, however, rejected values below 1 — so saving any change to the profile (a rename, a parsing tweak) was blocked until you raised the limit, which would then let exam questions trickle into daily reviews. A value of 0 is now valid when the limit is enabled and means "introduce no new cards". If you were forced to set 1 on your Exams profile, set it back to 0.

#### **Fixed: the demo exam deck could appear as a regular deck**

On the very first start after the update, the demo exam deck could be created and parsed before its profile assignment existed, leaving it a plain review deck without the "Start exam" entry. Creation now waits for the profile assignment. If your demo deck was affected, delete the file and run the "Create demo exam deck" command to recreate it.

#### **Fixed: turning on exam questions could silently affect the wrong profile**

Enabling "Exam questions" runs a quick check for already-reviewed cards that would change type before applying the setting. If you switched to a different profile while that check was still running, the setting could end up applied to whichever profile you'd switched to instead of the one you toggled it on — enabling exam parsing on a profile you never touched. The toggle now only ever applies to the profile it was changed on. If a profile you didn't intend to use for exams has "Exam questions" turned on, turn it back off in the profiles manager.
